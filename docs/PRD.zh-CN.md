# 海外社媒内容生成工作台 PRD

## 1. 文档信息

- 产品名称：海外社媒内容生成工作台
- 版本：V1.0
- 日期：2026-03-19
- 状态：已批准进入 MVP 开发

## 2. 产品概述

海外社媒内容生成工作台是一款面向海外社媒运营场景、按项目组织的 AI 内容生成 Web 应用。每个项目只对应一个品牌。用户可在项目中配置品牌上下文、维护目标账号列表，账号按平台归属并支持同一平台下存在多个账号；同时可按需导入近期已发布样本、上传新的素材，并生成适配不同平台账号场景的英文社媒文案。

MVP 阶段聚焦于内容生成能力。内容发布与社媒账号授权明确不纳入当前范围，后续将通过 Zapier 或 Buffer 等集成方式处理。

## 3. 背景

同时管理多个海外品牌的团队，往往需要从分散的素材中生成社媒文案。他们需要一套工作流，用于：

1. 清晰隔离不同品牌的上下文。
2. 复用历史内容以保持语气一致。
3. 快速将新的图片或视频素材转化为可直接使用的平台文案。
4. 针对不同平台产出不同结果，而不是每次都手动重写。

当前工作流通常分散在笔记工具、图片文件夹、表格和 AI 对话窗口之间，导致迭代缓慢、品牌语气不一致以及重复劳动。

## 4. 产品目标

### 4.1 商业目标

1. 建立一个可用的海外社媒 AI 文案生成 MVP。
2. 验证按项目管理品牌上下文，是否能提升内容质量与工作效率。
3. 为后续内容发布集成能力打下基础。

### 4.2 用户目标

1. 快速切换不同品牌。
2. 让每个品牌的内容风格保持独立。
3. 基于多图输入或单个视频生成英文社媒文案。
4. 按需使用近期已发布样本来引导语气与结构。
5. 获取适配 X、Instagram、TikTok 和 YouTube 账号场景的优化版本。

### 4.3 MVP 非目标

1. 自动发布到社媒平台。
2. 社媒账号 OAuth 或原生平台授权。
3. 排期日历。
4. 团队审批工作流。
5. 数据分析与报表。
6. 评论回复生成。
7. 单次任务上传多个视频。

## 5. 用户

### 5.1 主要用户

管理一个或多个品牌账号的海外社媒运营人员。

### 5.2 次要用户

1. 同时服务多个客户品牌的小型内容团队。
2. 自行运营品牌账号的创始人或市场人员。

## 6. 范围

### 6.1 范围内

1. 项目的创建、编辑、删除与切换。
2. 品牌级上下文信息存储。
3. 项目维度的目标账号配置，支持同平台多账号。
4. 通过链接导入或手动录入样本，且均为可选。
5. 图片与视频素材上传。
6. 基于品牌上下文、历史样本与素材的 AI 文案生成。
7. 按账号拆分的结果视图。
8. 生成历史与可复用记录。

### 6.2 范围外

1. 内容发布工作流。
2. 平台 API 凭证管理。
3. 用户权限与多人协作。
4. 计费能力。
5. 大规模媒体处理流水线。

## 7. 关键假设

1. 一个项目只对应一个品牌。
2. 历史内容样本是可选项。
3. MVP 阶段对外暴露的输出语言仅为英文。
4. 系统架构需为未来多语言扩展做好准备。
5. 用户单次生成请求中可上传多张图片或一个视频。
6. MVP 支持的平台为 X、Instagram、TikTok 和 YouTube。
7. 同一项目下允许存在同一平台的多个账号。
8. YouTube 输出为标题加简介，不包含社区帖文。
9. TikTok 输出仅为 Caption。

## 8. 核心概念

### 8.1 项目

项目是单个品牌的顶层工作空间，包含：

1. 品牌元信息。
2. 目标账号配置。
3. 历史内容样本。
4. 已上传素材。
5. 生成记录。

### 8.2 平台与账号

平台是账号所属的渠道类型，真正的文案生成目标是账号。MVP 平台包括：

1. X
2. Instagram
3. TikTok
4. YouTube

每个项目下可配置多个账号，多个账号也可以属于同一平台。每个账号都拥有独立的生成偏好，不代表已经连接原生平台账号。

### 8.3 历史内容样本

历史样本是用于塑造新文案风格的近期内容参考，可来自：

1. 用户提供的社媒链接。
2. 用户手动粘贴的文本内容。
3. 可选地绑定到某个具体账号。

样本保持为可选项，以避免阻塞冷启动使用。

### 8.4 素材

素材可以是：

1. 一组上传的图片。
2. 一个上传的视频。

系统会对素材进行分析，提取主题、主体、可见文字与内容线索。

### 8.5 生成任务

一次生成任务会绑定：

1. 一个项目。
2. 一组所选素材。
3. 当前项目上下文。
4. 零个或多个相关历史样本。
5. 一个或多个目标账号。

## 9. 核心用户流程

### 9.1 创建并配置项目

1. 用户创建项目。
2. 用户填写品牌名称与品牌描述。
3. 用户定义受众、语气、禁用词和默认语言。
4. 用户创建并配置一个或多个目标账号。
5. 用户保存项目。

### 9.2 导入历史样本

1. 用户进入项目内的样本库。
2. 用户可添加社媒链接、手动文本，或两者同时添加。
3. 系统保存来源平台、来源类型、可选账号归属和样本文本。
4. 导入后的样本可用于后续生成任务。

### 9.3 基于新素材生成文案

1. 用户选择一个项目。
2. 用户上传多张图片或一个视频。
3. 用户可选地查看当前项目的样本与设置。
4. 用户启动生成。
5. 系统分析素材。
6. 系统组合素材理解结果、品牌上下文与近期样本上下文。
7. 系统返回：
   - 通用草稿
   - 所选 X 账号草稿
   - 所选 Instagram 账号 Caption
   - 所选 TikTok 账号 Caption
   - 所选 YouTube 账号标题与简介
8. 用户编辑、复制或保存结果。

### 9.4 复用已生成内容

1. 用户打开生成历史。
2. 用户查看此前结果。
3. 用户复制或细化某条历史草稿。
4. 用户后续将最终文案导出到外部发布工作流。

## 10. 功能需求

### 10.1 项目管理

1. 系统应支持创建、编辑、删除与切换项目。
2. 系统应完全隔离项目数据。
3. 每个项目应存储：
   - 品牌名称
   - 品牌描述
   - 目标受众
   - 品牌语气
   - 默认输出语言
   - 禁用词
4. 系统应展示项目概览面板，包括样本数量、素材数量和近期生成记录。

### 10.2 账号配置

1. 系统应支持为每个项目维护多个目标账号。
2. 系统应支持同一平台下存在多个账号。
3. 每个账号的配置应至少支持：
   - 所属平台
   - 账号名称
   - 账号标识（如 handle）
   - 启用状态
   - 期望长度
   - CTA 偏好
   - Hashtag 偏好
   - 风格备注

### 10.3 样本库

1. 系统应支持从社媒链接导入样本。
2. 系统应支持手动录入样本文本。
3. 链接导入与手动录入均应为可选项。
4. 系统应存储样本元数据，包括：
   - 平台
   - 可选账号归属
   - 来源链接
   - 发布时间（若可获取）
   - 样本文本
   - 导入方式
5. 样本导入失败不应阻塞文案生成。

### 10.4 素材上传

1. 系统应支持上传多张图片。
2. 系统应支持上传一个视频。
3. 系统应阻止在一次生成任务中组合多个视频。
4. 系统应展示已上传素材的摘要信息。
5. 系统应保留素材与生成结果之间的关联关系。

### 10.5 AI 生成

1. 系统应基于上传素材生成英文社媒文案。
2. 系统应纳入项目级品牌上下文。
3. 系统应在有样本时按需纳入历史样本。
4. 系统应支持一次生成多个候选结果。
5. 系统应支持按账号输出结果，不同账号的输出格式由所属平台决定：
   - X：帖子正文
   - Instagram：Caption
   - TikTok：Caption
   - YouTube：标题 + 简介
6. 系统应支持生成后的二次处理动作，例如重写、缩短和加强 CTA。

### 10.6 结果管理

1. 系统应存储生成历史。
2. 系统应支持按账号查看结果。
3. 系统应支持保存前手动编辑。
4. 系统应支持复制单条输出结果。
5. 系统应支持标记首选版本。

## 11. 平台输出要求

### 11.1 X

- 输出形态：帖子正文
- 目标：简洁、有明确观点或清晰的信息密度
- 典型约束：短文本、直接钩子、可选 CTA

### 11.2 Instagram

- 输出形态：Caption
- 目标：更强的视觉叙事与品牌表达
- 典型约束：语气更顺滑、情绪 framing 更强、可选 hashtags

### 11.3 TikTok

- 输出形态：Caption
- 目标：适配短视频的简短、直接、原生感更强的配文
- 典型约束：直接钩子、语气轻快、易于快速阅读

### 11.4 YouTube

- 输出形态：标题 + 简介
- 目标：标题负责吸引点击，简介负责补充信息
- 典型约束：标题需易扫读，简介需提供上下文与 CTA

## 17. 提示词与生成策略草案

生成链路应组合以下四层：

1. 品牌上下文层
   - 品牌摘要
   - 受众
   - 语气
   - 禁用词
2. 账号上下文层
   - 所属平台
   - 账号名称 / handle
   - 账号级风格备注
   - 长度、CTA、Hashtag 偏好
3. 样本风格层
   - 近期样本
   - 偏好的结构
   - 重复出现的措辞模式
4. 素材理解层
   - 场景摘要
   - 主体对象
   - OCR 文本
   - 可能的使用场景

系统应先产出一版通用英文草稿，再将其转化为不同账号的结果。

## 18. 错误与空状态处理

1. 没有样本时：
   - 明确提示即使没有样本也可以正常生成
2. 链接解析失败时：
   - 保留手动样本流程，并给出非阻塞告警
3. 用户上传多个视频时：
   - 以清晰提示拒绝该操作
4. 未启用任何账号时：
   - 阻止生成，并提示用户至少启用一个账号

## 19. 非功能需求

1. 强项目级数据隔离。
2. 输入与输出之间可追溯。
3. 错误提示清晰且可恢复。
4. 架构为未来多语言扩展做好准备。
5. 输出结构可扩展，以支撑后续发布集成。

## 20. MVP 成功指标

1. 用户可以成功创建并切换项目。
2. 用户在没有样本的情况下也能完成生成流程。
3. 用户可以成功为所选目标账号生成输出结果。
4. 用户可以在不离开应用的情况下查看并复制生成内容。
5. 至少有一条生成记录可从历史中重新打开。
